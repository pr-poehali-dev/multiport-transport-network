# LogisticHub — Архитектура и план развития

## Текущая структура проекта

### Технологический стек
- **Frontend**: React + TypeScript + Vite
- **UI Components**: shadcn/ui
- **Стили**: Tailwind CSS
- **Шрифты**: IBM Plex Sans (интерфейс), Roboto (данные)
- **Иконки**: lucide-react через wrapper Icon

### Цветовая схема (B2B)
- **Primary**: `#0EA5E9` (голубой акцент)
- **Sidebar**: `#1A1F2C` (темный)
- **Background**: `#F9FAFB` (светло-серый)
- **Foreground**: темно-серый текст
- **Borders**: мягкие серые границы

## Структура данных

### Order (Заказ)
```typescript
interface Order {
  id: string;              // ORD-XXXX
  route: string;           // "Москва → Владивосток"
  status: OrderStatus;     // В пути | Доставлено | Ожидание | Задержка
  carrier: string;         // Название перевозчика
  progress: number;        // 0-100%
  eta: string;             // Ожидаемое время доставки
  createdAt: Date;
  updatedAt: Date;
}
```

### Notification (Уведомление)
```typescript
interface Notification {
  id: string;
  type: 'info' | 'warning' | 'success' | 'alert';
  title: string;
  message: string;
  time: string;
  read: boolean;
  relatedOrderId?: string;
}
```

### Partner (Партнер)
```typescript
interface Partner {
  id: string;
  name: string;
  rating: number;          // 0-5
  completedOrders: number;
  activeOrders: number;
  contactEmail: string;
  contactPhone: string;
  regions: string[];       // Регионы работы
}
```

## UI/UX принципы

### Дизайн-система
- **Минимализм**: Чистый интерфейс без лишних элементов
- **Информативность**: Вся важная информация на виду
- **Скорость**: Быстрый доступ к критичным данным
- **Профессионализм**: Строгий B2B стиль

### Компоненты
- Карточки с hover-эффектами
- Цветные бейджи для статусов
- Прогресс-бары для визуализации
- Иконки для быстрого распознавания
- Разделители для группировки информации

### Анимации
- Плавные переходы между состояниями
- Hover-эффекты на интерактивных элементах

## Примечания

### Приоритеты разработки
1. **Критичные функции**: детальная страница заказа, реальные данные из БД
2. **Важные функции**: разделы партнеров и маршрутов
